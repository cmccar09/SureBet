Step-by-step pipeline
1

Yesterday's Results — run automatically each morning

sl_results_fetcher.py

Matches pending DynamoDB picks against the SL fast-results page. Updates outcome = win / placed / loss + profit + SP odds.

⚡ DST: race times converted UTC → BST before matching SL local times.

python sl_results_fetcher.py
2

Today's Racecard — MUST run first

sl_racecard_fetcher.py

Fetches all declared runners from Sporting Life racecard. Writes racecard_cache.json (keyed by date → course → race → runners). Source of truth for declared field sizes — required by the S8 completeness gate.

python sl_racecard_fetcher.py
3

Convert Racecard to Workflow Format

sl_to_workflow_adapter.py

Converts racecard_cache.json → response_horses.json (the format complete_daily_analysis.py expects). Also merges Betfair decimal odds where available.

python sl_to_workflow_adapter.py
4

Live Betfair Odds Refresh OPTIONAL

betfair_odds_fetcher.py

Fetches live Betfair odds and injects them into response_horses.json. Without this, odds come from SL racecard (SP estimates only — no market_leader signal).

python betfair_odds_fetcher.py
5

Form Enrichment AUTO (imported)

form_enricher.py

Fetches last-6-race history per runner. Unlocks deep_form signals: course_win, distance_win, going_win, etc. Results cached in form_cache.json — do not delete, takes ~20 min to rebuild.

auto-invoked
6

Main Analysis + Pick Selection CORE

complete_daily_analysis.py

Scores every horse with the 7-factor model. Runs quality gates S1–S8. Selects top-5 picks. Saves ALL horses to DynamoDB SureBetBets.

Depends on: response_horses.json · racecard_cache.json · comprehensive_pick_logic.py

python complete_daily_analysis.py
7

Notify Picks

notify_picks.py

Sends WhatsApp notifications for today's top picks. Times shown in UK local time (BST/GMT), not UTC.

python notify_picks.py
8

Deploy API ONLY IF LOGIC CHANGED

_deploy_bpapi.py

Zips _bpapi_patched.py → lambda_function.py and deploys to AWS Lambda. Lambda serves DynamoDB picks to the frontend (read-only — no scoring in Lambda).

python _deploy_bpapi.py

Quality gates — _passes_quality_gates() in complete_daily_analysis.py
Gate Rule Action
S1 Non-market-backed picks: score ≥ 85 (with trainer/course-distance anchor) or ≥ 88 (no anchor) BLOCK
S2 At least one contextual anchor required (market, trainer, cd_bonus, unexposed, deep_form) BLOCK
S3 Age-padded picks (age_bonus ≥ 10) without market or trainer backing need score ≥ 92 BLOCK
S4 Score gap ≥ 8 vs 2nd-best horse in race (close races = coin flips) BLOCK
S5 Saturday: gap ≥ 15 required; 16+ runner fields additionally need market backing BLOCK
S6 Large field (16+ runners) without market backing: gap ≥ 12 BLOCK
S7 Novice / debut-heavy races where ≥ 60% of runners have ≤ 1 career run: skip entirely BLOCK
S8 Incomplete field — SL declared N runners but Betfair returned fewer:
● Missing 2+: always block
● Missing 1, not market leader: block
● Missing 1, market leader: warning only — allow through 		NEW ✓
⚠ Lesson (S8 — 2026-04-01): Kingcormac (Wincanton 15:20, Class 5) was our top scorer but a 6th runner Starlucky was absent from Betfair — the winner we never analysed. S8 would have blocked this pick automatically. racecard_cache.json is now cross-referenced on every pick.

complete_daily_analysis.py run directly

Deep form signals from Racing Post / SL. Cached in form_cache.json.

Going and weather condition signals, invoked by scoring engine.

trainer_form_stats.py imported

Trainer recent hot-streak data, feeds trainer_bonus into scoring.

betfair_odds_fetcher.py run directly

Live Betfair odds — optional but required for the market_leader signal.

DST / BST notes: All DB race_time values are stored as UTC. SL results pages show UK local time (BST = UTC+1 in summer). sl_results_fetcher.py converts UTC→BST via _to_uk_local_hhmm() before matching. notify_picks.py also converts times to local before sending WhatsApp messages.

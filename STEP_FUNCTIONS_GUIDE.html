What Step Functions Does Overview

Before Step Functions, the daily pipeline relied on a script running on a local PC. If the PC was off, a step failed silently, or the script was never started, picks were missed. Step Functions replaces this with a fully managed AWS orchestration layer that launches automatically via EventBridge — even if your laptop is off — and retries, checks, and reports on every step.

🕐

Runs on a Schedule

EventBridge triggers each pipeline at the configured UTC time. No manual intervention needed.

🔁

Auto-Retries on Failure

Each step has retry rules. Transient AWS or network errors automatically retry up to 2×.

🛑

Hard Stops on Bad Data

If validation fails (bad picks, 0 runners), the pipeline aborts — no junk is notified.

📊

Full Execution History

Every execution, step duration, input/output and error is visible in AWS Console.

📧

Evening Report Email

Automated HTML P&L report emailed via SES every night — no manual reports needed.

🧠

Nightly Self-Learning

Scoring weights are updated automatically from 7 days of settled results every night.

How Failures Are Handled Retry & Abort Logic

Automatic Retries

Every Lambda task in the state machines has a Retry block for transient errors. AWS automatically re-invokes the function before passing control to the Catch handler.

Lambda timeout / throttle / 500
→
Wait 30s → Retry (×2)
→
Backoff ×2
→
Catch → Pipeline Failed state

Validation Hard Stop

If surebet-validate finds a pick with empty runners, 0 score, or odds ≤ 1 — it raises an exception. Step Functions catches this and the WhatsApp notification step is never reached.

Analysis saves picks
→
Validate checks each pick
→
❌ Bad pick found → exception
→
Pipeline FAILED — no notification
Validate checks each pick
→
✅ All picks valid
→
Notify via WhatsApp

Non-Fatal Steps (Catch without Fail)

Some steps are important but must not kill the whole pipeline if they fail:

📱 WhatsApp Notify

If Twilio fails (outage, invalid creds), the pipeline still marks itself SUCCEEDED. You won't get a message but picks are still saved and visible on the UI.

🌙 Betfair Results Fetch

If the results aren't settled yet at 20:00, the pipeline continues to generate and send the evening report based on whatever outcomes are already in DynamoDB.

Data Flow Between Steps

Step Functions passes a JSON object from step to step. Each Lambda reads it, does work, and returns its own JSON — which gets merged in via ResultPath. No step can accidentally overwrite another's data.

// Step Functions state object as it flows through Morning pipeline
{
  "date": "2026-04-04",                    ← injected by InjectDate state
  "fetchResult": {
    "race_count": 42,
    "s3_key": "daily/2026-04-04/response_horses.json"  ← from surebet-betfair-fetch
  },
  "analysisResult": {
    "picks_count": 5                        ← from surebet-analysis
  },
  "validateResult": {
    "valid_picks": 5, "errors": []           ← from surebet-validate
  },
  "notifyResult": {
    "notifications_sent": 2, "skipped": false  ← from surebet-notify
  }
}

Day-to-Day Operations

Check if today's pipeline ran

python deploy_step_functions.py --status

Manually trigger the morning pipeline

aws stepfunctions start-execution --region eu-west-1 \
  --state-machine-arn arn:aws:states:eu-west-1:813281204422:stateMachine:SureBet-Morning \
  --input '{"date":"2026-04-05"}'

Redeploy Lambda code after a code change

python deploy_step_functions.py --lambdas

Full redeploy (infrastructure + code)

python deploy_step_functions.py

View execution logs in CloudWatch

aws logs tail /aws/lambda/surebet-analysis --region eu-west-1 --since 1h
aws logs tail /aws/lambda/surebet-betfair-fetch --region eu-west-1 --since 1h
